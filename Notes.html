Sockets

  • a socket is an endpoint for sending or receiving data across a computer network. It is a combination of an IP address and a port number, which together uniquely identify a connection between two devices.
  • sockets are used to communicate between a client and a server
  • IP Address: This is the address of the device on the network.
    Port Number: This is a specific endpoint on the device for communication.

Protocols

  • Protocols are a set of rules and conventions for communication between network devices. They ensure that data is transmitted accurately and reliably.
  • ( Transmition Control Protcols )TCP :- It is one of the main protocols of the Internet Protocol (IP) suite. TCP provides reliable, ordered, and error-checked delivery of a stream of data between applications running on hosts communicating via an IP network.
  • ( User Datagram Protocol )UDP :- It is one of the core members of the Internet Protocol (IP) suite. Using UDP, programs on networked computers can send short messages sometimes known as datagrams (using Datagram Sockets) to one another.
  • WebRTC (Real time communicate):- It is a Protocol tha help us to direcly communicate with client to client. Without any Middleware(server).
  • WebRTC is a free, open-source project that provides web browsers and mobile applications with real-time communication via simple application programming interfaces (APIs). It allows audio and video communication to work inside web pages by allowing direct peer-to-peer communication, eliminating the need to install plugins or download native apps.

Working Of WebRTC :-

  • Steps 1 :- Signaling :- It is the process of coordinating communication between WebRTC clients. It is not defined by the WebRTC standard, so you can use any technology you like to implement it.
    Steps 2 :- ICE (Interactive Connectivity Establishment) :- It is a framework to allow your web browser to connect with peers. It is a collection of protocols that help WebRTC clients discover and connect with each other.
    Steps 3 :- Security (DLTS & SRTP) :- It is a protocol that encrypts and secures the data that is sent between WebRTC clients.
    Steps 4 :- NAT Traversal :- It is a technique that helps WebRTC clients to connect with each other even if they are behind a NAT (Network Address Translation) router.

WebRTC API

  • WebRTC (Web Real-Time Communication) is a technology that enables peer-to-peer audio, video, and data sharing between browsers without requiring an intermediary server. It is commonly used for applications such as:
    - Video conferencing
    - Voice calls
    - File sharing
    - Real-time chat applications
    - Online gaming
    The WebRTC API provides several key components:
    1. `RTCPeerConnection`: Manages the connection between peers, handling the negotiation of the connection and the transmission of media and data.
    2. `RTCDataChannel`: Enables the transmission of arbitrary data directly between peers.
    3. `getUserMedia()`: Captures audio and video from the user's device.
    4. `RTCIceCandidate`: Represents a candidate network path for connecting peers.
    5. `RTCSessionDescription`: Describes one end of a connection, including the media format and network information.
    Example use cases:
    - Building a video chat application where users can see and hear each other in real-time.
    - Creating a multiplayer game that requires real-time data exchange between players.
    - Developing a file-sharing service that allows users to send files directly to each other without uploading to a server.

  • MCU (Multi Point Control Units ):- An MCU (Multipoint Control Unit) is a server that facilitates multipoint video conferencing by managing the connections between multiple participants. It acts as a central hub that receives video and audio streams from each participant, processes them, and then redistributes the combined streams back to all participants. This allows multiple users to communicate in a single session, even if they are on different networks or using different devices.
